Darkmesh

Decentralized command-and-control for environments where the network is jammed and the coordinator just got killed.

// The Problem

The DoD's nightmare is a near-peer fight where adversaries jam Starlink and sever cloud links — and nodes get hit, batteries die, bags get dropped. Most C2 stacks have a single point of failure even after you cut the cloud. Darkmesh survives both.

// Architecture

TIER 3 — Palantir Foundry / CASK / OSDK
         ▲ store-and-forward when uplink returns
         │ AIP Logic: severity + SITREP + cue draft
         │
TIER 2 — Jetson Orin Nano + 2× Raspberry Pi 4B
         ▲ private LAN (no internet, no satellite)
         │ Local LLM on every node (Gemma 4 e2b / Gemma 3)
         │ Raft-style coordinator election
         │ Gossip-replicated world state
         │
TIER 1 — Sensors: camera, microphone, RFID, location
         ↓ multimodal fusion → CASK evidence bundles

// What's Different

No single point of failure

Kill the coordinator mid-pitch. Another node assumes command in under one second via Raft heartbeat consensus. No data loss. No operator awareness gap.

No cloud assumptions

Gemma 4 e2b runs on the Jetson, Gemma 3 on each Pi. Multimodal sensor fusion happens locally. When uplink returns, evidence reconciles to Palantir Foundry.

Real partner integrations

Palantir AIP for severity classification + SITREP drafts. Foundry OSDK for ontology mirror + writeback. Danti for geospatial enrichment. OpenAI Codex for build velocity.

Human-in-the-loop, always

Counter-UAS cueing only — no autonomous engagement, no kill chain automation. Every actionable output is policy-gated and operator-acknowledged.
